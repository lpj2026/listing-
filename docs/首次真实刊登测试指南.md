# 首次真实刊登测试指南

> 目标：在阿里云白名单 IP 上，用测试 MSKU 跑通一次完整链路  
> `填表 → 领星 publish → 亚马逊 → Listing 同步 → SKU 配对`

---

## 一、测试前检查清单

### 1.1 环境与凭证

- [ ] 领星后台已配置 AppID / AppSecret（`.env` 中 `LINGXING_APP_ID`、`LINGXING_APP_SECRET`）
- [ ] 阿里云服务器 `8.137.177.25` 已在领星 **IP 白名单**
- [ ] 服务器 `.env` 已配置，且 `APP_PORT=8001`（与补货系统 8000 分开）
- [ ] 刊登系统可访问：`http://8.137.177.25/listing/create-product`（或本地 `http://127.0.0.1:8001/create-product` 且 IP 在白名单）
- [ ] Python 依赖已安装：`pip install requests pycryptodome`

### 1.2 店铺与权限

- [ ] 选用 **非主力 / 测试店铺**（美国站优先，类目限制少）
- [ ] 打开页面后，店铺下拉显示「数据来源：领星 API」（非 Mock）
- [ ] 分类选择器显示「数据来源：领星 API」
- [ ] 该店铺在亚马逊后台已授权 SP-API（领星侧正常同步 Listing 即可）

### 1.3 测试 SKU 规范

- [ ] MSKU 命名：`TEST-YYYYMMDD-001`（例：`TEST-20260609-001`）
- [ ] **全局唯一**，不与现有 MSKU 重复
- [ ] 测试完成后计划下架或删除测试 Listing
- [ ] 本地 SKU（领星 ERP）已准备好，用于配对（例：`测试品名 OTII-230722-0828`）

### 1.4 商品资料准备

- [ ] 已在亚马逊申请对应类目的 **UPC/GTIN 豁免**，或手头有有效 UPC
- [ ] 主图 URL 为 **公网可访问的 HTTPS 链接**（非本地文件、非 base64）
- [ ] 标题、品牌、制造商、价格、数量、描述已准备（英文）
- [ ] 已选 **末级分类**，并确认 `productType` 已自动填入

## 二、预检步骤（约 10 分钟）

按顺序执行，全部通过后再填表刊登。

### 步骤 1：验证 API 连通

在服务器或本机（IP 在白名单时）：

```powershell
cd d:\Product\backend
python -c "from seller_service import list_stores; r=list_stores(); print(r['source'], len(r['data']), r['message'])"
```

**预期**：`lingxing 133 数据来源：领星 API...`

### 步骤 2：验证分类 API

浏览器打开刊登页 → 选店铺 → 点「选择分类」→ 能展开真实分类树。

**预期**：弹窗底部显示「数据来源：领星 API」，非「演示数据」。

### 步骤 3：验证 Schema

选到末级分类后，产品属性区应加载对应 `productType` 字段。

**预期**：Toast 显示「数据来源：领星 Schema API」或已加载属性模板。

### 步骤 4：部署到阿里云（若尚未部署）

```powershell
cd d:\Product
$env:REMOTE_SSH_PASSWORD="你的root密码"
python deploy_to_aliyun.py
```

**预期**：部署脚本无报错，可打开 `http://8.137.177.25/listing/create-product`。

## 三、正式测试步骤

### 步骤 1：打开页面并选店

1. 访问 `http://8.137.177.25/listing/create-product`
2. 选择 **测试店铺 + 美国站**
3. 确认控制台无 403 / Failed to fetch

### 步骤 2：选择分类

1. 点击「选择分类」
2. 逐级选到 **末级分类**（`hasChildren=0`）
3. 确认 `#productType` 有值（如 `AUTO_PART`）
4. 确认分类路径已显示

### 步骤 3：填写基本信息

| 字段 | 测试值示例 | 必填 |
|------|-----------|------|
| 产品标题 | `Test Product DO NOT BUY` | ✅ |
| Parent SKU | `TEST-20260609-001` | ✅（作 MSKU） |
| UPC 豁免 | 若已申请豁免选「是」，否则填真实 UPC | ✅ |
| 品牌 | `Generic` 或真实品牌 | ✅ |
| 制造商 | 同品牌或真实制造商 | ✅ |
| 本地 SKU | 领星 ERP 中的本地 SKU | 建议填（自动配对） |

### 步骤 4：填写产品与价格

| 字段 | 说明 |
|------|------|
| 价格 | 填测试价，如 `9.99` |
| 产品数量 | FBM 填库存，如 `10` |
| 产品描述 | 英文描述，至少一段 |
| 五点描述 | 至少 1 条 bullet |
| 主图 | 粘贴 **公网 HTTPS 图片 URL** |

### 步骤 5：保存草稿（建议）

1. 点「保存」
2. 右侧草稿列表出现记录
3. 刷新页面 →「恢复」草稿，确认字段完整

### 步骤 6：提交刊登

1. 点 **「保存并发布」**
2. 观察 Toast：`已提交领星刊登，系统将自动查询结果`
3. 右侧「刊登任务」出现新任务，状态 `已提交领星，自动轮询中`

### 步骤 7：等待刊登结果（自动，约 10 分钟内）

任务状态变化：

```
已提交领星，自动轮询中 → 刊登处理中 → 刊登成功，等待 Listing 同步
```

| 状态 | 含义 | 操作 |
|------|------|------|
| 刊登处理中 | 亚马逊处理中 | 等待，或点「查刊登」 |
| 刊登失败 | 亚马逊拒绝 | 看「失败原因」，改字段后重新发布 |
| 轮询超时 | 15 分钟无结果 | 点「查刊登」手动查 |

**失败时常见原因**：

- 缺少该类目必填属性（Schema 未填全）
- 图片 URL 无法访问
- UPC 无效或未豁免
- `item_type_keyword` 与分类不匹配

### 步骤 8：等待 Listing 同步（自动，15 分钟～1 小时）

刊登成功后：

```
刊登成功，等待 Listing 同步 → Listing 已同步 / 已完成（含配对）
```

| 状态 | 含义 |
|------|------|
| 等待 Listing 同步 | 系统在查领星 Listing 是否出现 MSKU |
| Listing 已同步 | MSKU 已出现，但未配对（未填本地 SKU） |
| 已完成 | Listing 已同步且 SKU 已配对 |

可点「查Listing」手动加速检查。

### 步骤 9：验证结果（三个地方）

- [ ] **刊登任务**：状态 `已完成` 或 `Listing 已同步`，有 ASIN（若返回）
- [ ] **领星 ERP → Listing**：能搜到 `TEST-20260609-001`
- [ ] **亚马逊 Seller Central → 管理所有库存**：MSKU 存在（可能有延迟）

### 步骤 10：验证 SKU 配对

- [ ] 领星 Listing 中 MSKU 已关联本地 SKU
- [ ] 任务显示「Listing 已同步，SKU 已配对」

## 四、验收标准（首次测试通过条件）

| # | 检查项 | 通过 |
|---|--------|------|
| 1 | publish 提交返回 `record_unique_id` | ☐ |
| 2 | 刊登结果轮询为成功（非失败/超时） | ☐ |
| 3 | 领星 Listing 能查到测试 MSKU | ☐ |
| 4 | 亚马逊后台能看到该 MSKU（可选，有延迟） | ☐ |
| 5 | MSKU 与本地 SKU 配对成功（若已填本地 SKU） | ☐ |

**最低通过线**：第 1、2、3 项全部通过。

## 六、故障排查速查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 店铺/分类显示 Mock | IP 不在白名单或未配凭证 | 在阿里云运行或加白名单 |
| 提交报 403 | 出口 IP 未加白名单 | 确认从 `8.137.177.25` 发出 |
| 提交报「缺少 store_id」 | 未选店铺 | 重新选店铺账号+站点 |
| 提交报「缺少 attributes」 | 表单几乎为空 | 至少填标题、品牌、价格 |
| 刊登失败 | 亚马逊 Schema 校验 | 读 `failure_reason`，补属性 |
| Listing 长时间不同步 | 正常延迟 | 等 1 小时；领星后台手动同步 |
| 配对失败 | seller_id 缺失或本地 SKU 错误 | 确认店铺来自领星 API；核对本地 SKU 名称 |

## 七、推荐测试商品策略

**第一次建议**：选已熟悉、已申请 UPC 豁免的 **简单标品类目**（如部分汽配配件），字段相对固定。

**不建议第一次就测**：

- 多变体（父子 SKU）
- 需 A+ 或复杂属性的大类目
- 主力店铺、真实库存 SKU

## 八、相关文件与接口

| 用途 | 路径 |
|------|------|
| 本地数据 | `data/tasks.json`、`data/drafts.json` |
| 刊登提交 | `POST /api/publish` |
| 查刊登结果 | `GET /api/publish/result/{task_no}` |
| 查 Listing | `POST /api/tasks/{task_no}/sync` |
| SKU 配对 | `POST /api/tasks/{task_no}/pair` |
| 领星 publish | `/listing/publish/openapi/amazon/product/publish` |
| 领星 Listing 查 | `/erp/sc/data/mws/listing` |
| SKU 配对 | `/erp/sc/storage/product/link` |
